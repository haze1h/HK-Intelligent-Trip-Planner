# Planner Agent + Budget 接口说明文档（2026.3.19更新版）
（HK Intelligent Trip Planner 项目）

本文件用于说明当前系统中 **Planner Agent / Budget Tool / Critic Agent** 的接口定义与数据流，确保各模块开发与集成一致。

---

# 🧠 1. 系统整体架构（最新）

```mermaid
flowchart TD

A[UserRequest 用户输入] --> B[Budget Tool<br/>estimate_fixed_costs]
B --> C[Activity Budget<br/>activity_budget_hkd]

C --> D[Planner Agent (LLM)<br/>只选择活动名称]
D --> E[Resolver / Normalizer<br/>回填真实数据]

E --> F[Budget Tool<br/>estimate_budget]
F --> G[Critic Agent<br/>质量评估]

G --> H[Final Output]
````

---

# ⚠️ 2. 核心架构更新（必须理解）

## Planner Agent 现在：

```text
只负责：选择活动（activity_name）
```

不再负责：

* ❌ cost计算
* ❌ duration计算
* ❌ total预算
* ❌ 是否超预算判断

---

## 新增关键模块

### Resolver / Normalizer（关键）

负责：

* activity_name → dataset查找
* 回填真实字段：

  * area
  * category
  * cost_hkd
  * duration_hours
* 计算：

  * daily_cost
  * total_cost

👉 **系统稳定性的核心**

---

# 💰 3. 预算逻辑（必须理解）

## 3.1 用户输入

```text
total_budget_hkd
```

表示：

```text
整趟旅行总预算（包含吃住行 + 活动）
```

---

## 3.2 系统内部拆分

```text
activity_budget = total_budget - fixed_cost_estimate
```

---

## 3.3 Planner 只使用：

```text
activity_budget_hkd
```

---

## ⚠️ 重要规则

Planner：

* ❌ 不使用 total_budget
* ✅ 只使用 activity_budget
* ❌ 不计算总价

---

# 🧩 4. 模块职责划分

| 模块            | 职责              |
| ------------- | --------------- |
| Budget Tool   | 固定成本估算 + 最终预算核算 |
| Planner Agent | 选择活动            |
| Resolver      | 补全真实数据 + 计算费用   |
| Critic Agent  | 评估行程质量          |
| Frontend      | 展示              |

---

# 🔧 5. Planner 接口

## 调用入口

```python
PlannerAgent.plan(user_request, activities)
```

---

## 内部流程

```text
plan():
  → estimate_fixed_costs()
  → build_prompt()
  → LLM generate
  → extract JSON
  → validate minimal structure
  → resolve_and_normalize()  ←（关键变化）
  → return planner_output
```

---

# 📥 6. 输入说明

## 6.1 UserRequest

```python
UserRequest(
    destination: str,
    days: int,
    total_budget_hkd: float,
    preferences: List[str],
    pace: str = "moderate",
    must_include: Optional[List[str]] = None,
    avoid: Optional[List[str]] = None,
    travelers: int = 1,
    budget_style: Optional[str] = None
)
```

---

## 6.2 Activity 数据

```python
Activity(
    name: str,
    category: str,
    area: str,
    best_time: str,
    duration_hours: float,
    cost_hkd: float,
    indoor: bool,
    tags: List[str]
)
```

---

## ⚠️ 强约束

* ❌ 不允许修改 dataset
* ❌ 不允许新增活动
* ❌ LLM 不能编造活动

---

# 🔐 7. 强约束（新版本）

Planner 必须：

* 只使用 dataset 中活动
* 不允许 hallucination
* 若无法匹配偏好 → 忽略
* 不输出 cost / total
* 不重复活动
