# Planner Agent + budget 接口说明文档（2026.3.18更新版）
（HK Intelligent Trip Planner 项目）

本文件用于说明 Planner Agent 在引入 **Budget Tool** 后的接口、数据流与使用规范，确保后续组员在开发 **Critic Agent、Budget Tool、Frontend 或系统集成模块** 时能够正确对接。

---

# 1. 模块职责（重要更新）

Planner Agent 的职责：

```

只负责生成 itinerary（活动安排）

```

不再负责：

- ❌ 总预算计算
- ❌ 固定成本估算
- ❌ 总预算是否超支判断

---

# 2. 系统整体数据流（必须理解）

当前系统采用模块化架构：

```

UserRequest
↓
Budget Tool (estimate_fixed_costs)
↓
Planner Agent (使用 activity_budget)
↓
Budget Tool (estimate_budget)
↓
Critic Agent（未来扩展）
↓
Frontend

```

---

# 3. 核心预算逻辑（非常重要）

## 3.1 用户预算定义

用户输入：

```

total_budget_hkd

```

表示：

```

整趟旅行总预算（包含吃、住、行、活动）

```

---

## 3.2 预算拆分公式

系统内部计算：

```

activity_budget = total_budget - fixed_cost_estimate

```

Planner 只使用：

```

activity_budget_hkd

```

---

## 3.3 budget_style（预算档位）

Budget Tool 会根据以下信息推断预算档位：

- 用户 preferences（如 budget / luxury）
- 人均日预算
- 用户是否显式指定

可选值：

```

economy / standard / premium

````

不同档位影响：

- 住宿成本
- 餐饮成本
- 交通成本

⚠️ 这是启发式估算（heuristic），不是精确价格。

---

# 4. Planner 调用接口

唯一正式入口：

```python
PlannerAgent.plan(user_request, activities)
````

---

## 4.1 内部执行流程

```
plan():
  → estimate_fixed_costs()        # Budget Tool
  → build_prompt()
  → LLM generate
  → extract JSON
  → validate structure
  → normalize cost（代码重算）
  → return planner_output
```

---

# 5. 输入说明

## 5.1 UserRequest

定义在：

```
planner_agent/schemas.py
```

结构：

```python
UserRequest(
    destination: str,
    days: int,
    total_budget_hkd: float,
    preferences: List[str],
    pace: str = "moderate",
    must_include: Optional[List[str]] = None,
    avoid: Optional[List[str]] = None,
    travelers: int = 1,
    budget_style: Optional[str] = None
)
```

---

## 字段说明

| 字段               | 含义       |
| ---------------- | -------- |
| destination      | 目的地      |
| days             | 行程天数     |
| total_budget_hkd | 总预算      |
| preferences      | 用户偏好     |
| pace             | 行程节奏     |
| must_include     | 必须包含     |
| avoid            | 避免       |
| travelers        | 人数       |
| budget_style     | 预算档位（可选） |

---

## 5.2 Activity 数据

来源：

```
data/hk_activities.json
```

结构：

```python
Activity(
    name: str,
    category: str,
    area: str,
    best_time: str,
    duration_hours: float,
    cost_hkd: float,
    indoor: bool,
    tags: List[str]
)
```

⚠️ 字段不可修改

# 8. itinerary 结构要求

每一天必须包含：

```
morning / afternoon / evening
```

每个 slot：

| 字段                 | 含义   |
| ------------------ | ---- |
| activity_name      | 活动名称 |
| area               | 地区   |
| category           | 类型   |
| estimated_cost_hkd | 活动费用 |
| duration_hours     | 时长   |
| reason             | 推荐理由 |

---

## 强制约束

* itinerary 长度必须 = days
* 每天必须 3 个 slot
* 不允许重复活动

---

# 9. 关键规则（必须遵守）

Planner 必须：

* 只使用 activity_budget
* 不包含固定成本（吃住行）
* 不信任 LLM 的 cost（必须重算）
* 严格输出 JSON

# 12. 常见错误（重要）

## ❌ 错误 1：Planner 使用 total_budget

→ 应该使用 activity_budget

---

## ❌ 错误 2：活动费用包含吃住行

→ 不允许

---

## ❌ 错误 3：少一天 itinerary

→ 必须等于 days

---

## ❌ 错误 4：相信 LLM 成本

→ 必须代码重算

---

# 13. 调试建议

检查：

* itinerary 长度
* 是否重复活动
* daily_cost 是否正确
* 是否超 activity_budget
